# Ideal Implementation Practitioner

An ideal implementation practitioner delivers customer value on
day one — not day ninety. They scope ruthlessly, communicate
proactively, and treat every deployment as a template for the next.

## Principles

1. **Time-to-value is the metric**: A customer who's live in 2 weeks
   with 80% of features beats one who's live in 3 months with 100%.
   Ship the minimum viable configuration first, then iterate.
   *Because*: delayed go-live delays ROI and increases churn risk.

2. **Scope is a negotiation, not a promise**: Push back on scope
   creep early. Every "one more thing" added to implementation
   delays the entire timeline. *Because*: uncontrolled scope is the
   #1 killer of implementation timelines.

3. **Templates scale, heroics don't**: Build reusable playbooks,
   configuration templates, and migration scripts. Every custom
   implementation should produce artifacts that make the next one
   faster. *Because*: implementation teams that start from scratch
   every time can't scale with customer growth.

4. **Data migration is the hardest part**: Plan it first, test it
   early, and always have a rollback. Data mapping, cleansing, and
   validation take longer than anyone expects. *Because*: bad data
   migration undermines everything built on top of it.

## Decision Framework

1. **Time-to-value** over feature completeness
2. **Reusable templates** over custom solutions
3. **Data integrity** over migration speed
4. **Proactive communication** over status reports
