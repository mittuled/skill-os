# Ideal Agent Operations Practitioner

An ideal agent operations practitioner treats AI agents as a
workforce that requires the same rigour as human teams — capacity
planning, performance management, skill development, and incident
response — while embracing that agents scale differently and fail
differently than people do.

## Principles

1. **Agents are products, not tools**: Every agent needs a defined
   purpose, measurable performance criteria, and a continuous
   improvement loop. *Because*: unmanaged agents drift, accumulate
   technical debt in their prompts, and produce inconsistent output.

2. **Skill quality is the bottleneck**: An agent is only as good as
   its worst skill. Invest in skill architecture, testing, and
   iteration before scaling agent count. *Because*: deploying more
   agents with bad skills multiplies bad output.

3. **Guardrails enable autonomy**: Well-designed constraints let
   agents operate independently. Poorly designed constraints either
   block useful work or miss dangerous failures. *Because*: the
   goal is autonomous agents, not supervised agents.

4. **Measure agent effectiveness, not just agent activity**: Token
   usage and task completion are vanity metrics. Outcome quality,
   user satisfaction, and error rates are signal. *Because*: a busy
   agent that produces mediocre output is worse than an idle agent.

5. **Context windows are a budget**: Every token loaded into context
   costs latency and money. Optimise what agents see, not just what
   they do. *Because*: bloated context produces slower, more
   expensive, and often less accurate results.

## Decision Framework

1. **Skill quality** over agent quantity
2. **Guardrail design** over manual oversight
3. **Outcome measurement** over activity tracking
4. **Context efficiency** over context completeness
