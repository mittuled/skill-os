# Scoring Rubric: moat-analyzer-tech

Evaluates the strength and durability of a company's technical competitive moat across data advantages, replication barriers, network effects, integration depth, and threat exposure.

## Criteria

| # | Criterion | Weight | Description |
|---|-----------|--------|-------------|
| 1 | Data Advantage | 30% | Proprietary data assets that compound over time and cannot be acquired or synthesized by competitors |
| 2 | Replication Barrier | 25% | Time, capital, and talent required for a well-funded competitor to replicate the core technical capability |
| 3 | Network Effects | 20% | Degree to which each additional user or data point increases value for all others |
| 4 | Integration Depth | 15% | Switching cost generated by deep integrations into customer workflows, data stores, or infrastructure |
| 5 | Open Source / Commoditization Risk | 10% | Exposure to open-source projects or platform changes that could erode technical differentiation |
| **Total** | | **100%** | |

## Scale

Each criterion scored **0–10**: 0 = no moat evidence / critical exposure, 5 = present with significant gaps or risks, 10 = durable, compounding, defensible

**Composite Score** = Σ (criterion score × weight)

## Grade Bands

| Grade | Composite Score | Label | Description | Recommended Action |
|-------|----------------|-------|-------------|-------------------|
| A+ | 9.0–10.0 | Exceptional | Multiple compounding moat sources; replication estimated > 3 years and $50M+; deep integration lock-in | Maintain architecture; publish technical leadership content; actively widen moat |
| A | 8.0–8.9 | Strong | Clear data or integration moat; 18–36 month replication window; some network effect present | Invest in moat-strengthening features; monitor open-source landscape quarterly |
| B | 7.0–7.9 | Good | One solid moat source; 12–18 month replication window; limited network effects | Identify second moat source; accelerate data flywheel; assess platform dependencies |
| C | 5.0–6.9 | Adequate | Moat thin or single-dimensional; replication feasible within 12 months with adequate funding | Urgent moat-building sprint; re-evaluate architectural choices; assess acquisition options |
| D | 3.0–4.9 | Weak | No durable moat identified; features replicable in weeks to months; open-source near-parity | Pivot technical strategy; consider repositioning; moat is not defensible as-is |
| F | 0.0–2.9 | Failing | Pure commodity capability; direct open-source equivalents exist; no switching costs | Do not claim technical differentiation; rebuild from moat-first architectural principles |

## Signal Tables

### Data Advantage

| Score Range | Observable Evidence |
|-------------|-------------------|
| 9–10 | 12+ months of proprietary behavioral or transactional data with no public equivalent; data improves core model/algorithm performance by > 20% vs. synthetic alternatives; data acquisition by competitor requires years of market presence |
| 7–8 | 6–12 months of proprietary data; measurable quality improvement vs. public datasets; competitor could acquire similar data within 18 months at significant cost |
| 5–6 | Some proprietary data but supplemented heavily by public sources; competitor could construct comparable dataset within 12 months |
| 3–4 | Data advantage claimed but unquantified; primarily aggregated from public or purchased sources; competitor can buy equivalent data today |
| 0–2 | No proprietary data; model or algorithm trained on public datasets only; identical capability reproducible from open-source components |

### Replication Barrier

| Score Range | Observable Evidence |
|-------------|-------------------|
| 9–10 | Estimated replication time > 3 years and > $50M investment; requires scarce domain expertise (< 500 practitioners globally); regulatory or patent protection present |
| 7–8 | Estimated replication time 18–36 months; requires $10–50M investment; specialized team of 20+ senior engineers; significant infrastructure buildout required |
| 5–6 | Estimated replication time 6–18 months; requires $1–10M; well-understood architecture; replication feasible for Series B+ competitor |
| 3–4 | Replication possible in 3–6 months by a 5-person team; architecture is documented in public papers or open-source repos |
| 0–2 | Replicable in days to weeks using available open-source components; tutorial-level implementation exists publicly |

### Network Effects

| Score Range | Observable Evidence |
|-------------|-------------------|
| 9–10 | Each new user directly increases value for all existing users (data, social, or marketplace effect); product unusable or significantly degraded with zero users; network effect demonstrated in retention and engagement metrics |
| 7–8 | Network effect present but indirect; data or content produced by users improves the product for others; measurable retention difference between low-network and high-network users |
| 5–6 | Weak or claimed network effects; marginal value improvement per additional user; not yet demonstrated at scale |
| 3–4 | No meaningful network effect; product value is identical for user 1 and user 1,000,000 |
| 0–2 | Negative network effects present (congestion, noise, quality degradation at scale) |

### Integration Depth

| Score Range | Observable Evidence |
|-------------|-------------------|
| 9–10 | Bidirectional data sync with 5+ customer systems; replacement requires multi-month migration project; customer data stored in proprietary format with no export API; embedded in customer's critical workflow |
| 7–8 | Integrated with 2–4 customer systems; replacement requires 1–3 month migration; switching cost includes data migration and retraining |
| 5–6 | Single integration point; standard export formats available; switching requires 2–4 weeks of effort |
| 3–4 | Shallow integration; data can be exported in < 1 day; competitor onboarding requires minimal migration effort |
| 0–2 | No integration; product is standalone with standard import/export; zero switching cost beyond retraining |

### Open Source / Commoditization Risk

| Score Range | Observable Evidence |
|-------------|-------------------|
| 9–10 | No open-source project within 3 years of feature parity; core algorithm is proprietary with no published equivalent; platform dependencies are diversified |
| 7–8 | Open-source alternatives exist but are 18–36 months behind; company contributes to ecosystem and maintains lead; no single platform dependency > 30% of capability |
| 5–6 | Open-source alternatives 6–18 months behind; active community narrowing the gap; some platform concentration risk |
| 3–4 | Open-source project at 70–90% feature parity; community growing faster than company's development pace; platform dependency creates existential risk if policy changes |
| 0–2 | Direct open-source equivalent exists and is actively maintained; company's differentiation is wrapper or UX only; platform risk is unmitigated |
