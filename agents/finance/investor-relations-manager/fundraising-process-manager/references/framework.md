# Framework: fundraising-process-manager

Defines the end-to-end fundraising process architecture including timeline structure, pipeline management mechanics, diligence coordination, and term sheet analysis.

## Fundraising Process Phases

### Phase 1: Preparation (6–8 weeks before first investor meeting)

| Task | Owner | Deliverable |
|------|-------|-------------|
| Define raise parameters (amount, valuation, round structure) | CEO + CFO | Term sheet anchor |
| Update financial model with investor projections | FP&A | Investor-ready model |
| Refresh pitch deck | CEO + design | Investor deck v1 |
| Update data room | IR Manager | Complete data room |
| Build target investor list (50–100 firms) | IR Manager | Tiered target list |
| Map warm introduction paths | CEO + IR Manager | Intro request list |
| Brief existing investors on process | IR Manager | Existing investor email |

### Phase 2: Outreach and First Meetings (4–6 weeks)

Target: 20–30 first meetings in a compressed 4-week window.

| Activity | Standard | Notes |
|---------|---------|-------|
| Introduction method | Warm intro preferred; cold email acceptable | Track conversion rate by introduction type |
| Outreach timing | Send all outreach in the same 2-week window | Creates artificial simultaneity |
| First meeting format | 45–60 minute pitch + Q&A | CEO leads; IR Manager takes notes and tracks next steps |
| Post-meeting turnaround | Send follow-up summary within 24 hours | Captures commitment while momentum is high |
| Debrief cadence | CEO + IR after every meeting | Update deck and FAQs within 48 hours of identifying a pattern |

### Phase 3: Diligence (3–6 weeks per investor)

| Activity | SLA | Notes |
|---------|-----|-------|
| Data room access grant | Within 24 hours of investor confirming interest | Never grant before first meeting |
| Diligence request response | 48-hour SLA per item | Never let a request sit >72 hours |
| Reference call scheduling | Within 5 business days of request | Pre-brief all references |
| Follow-up questions | 24-hour response | Prioritize over other IR tasks during active diligence |

### Phase 4: Term Sheet and Negotiation (2–4 weeks)

| Activity | Best Practice |
|---------|---------------|
| Term sheet evaluation | Review all term sheets in parallel; do not accept the first one under artificial pressure |
| Key terms to negotiate | Pre-money valuation, liquidation preference structure (participating vs. non-participating), board composition, option pool size and timing of top-up, pro-rata rights, anti-dilution (broad-based weighted average is standard) |
| Board composition standard | Seed: 2 founders + 1 lead investor + 1 independent. Series A: 2 founders + 1 lead investor + 1 independent. Series B+: negotiate for independent director majority |
| Protective provisions | Expect: liquidation, merger/sale approval, new share class issuance. Resist: veto on hiring/firing CEO, budget approval requirements |

### Phase 5: Close (4–6 weeks)

| Activity | Owner | SLA |
|---------|-------|-----|
| Legal document drafting | Outside counsel | 2 weeks from term sheet signing |
| Investor legal review | Investor's counsel | 1-2 weeks |
| Closing conditions checklist | IR Manager | Track daily until all conditions satisfied |
| Wire receipt confirmation | CFO | Same day as expected close |
| Cap table update | IR Manager | Within 24 hours of wire receipt |
| Closing announcement | CEO | Within 1 week of close (if public) |

## Investor Pipeline Tracker Structure

| Field | Description |
|-------|-------------|
| Investor name / firm | Full name |
| Stage | Outreach / First meeting / Partner meeting / Diligence / Term sheet / Closed / Passed |
| Last contact date | Most recent meaningful interaction |
| Next step | Specific action with owner and due date |
| Valuation expectation | Any signal on valuation from conversations |
| Check size interest | Indicated range |
| Decision timeline | When they expect to make a decision |
| Pass reason | If passed — why (stage too late, sector not a fit, valuation gap, etc.) |
| Lead potential | Yes / No / Possible — based on conversation signals |
