# Framework: prd-assembler

Assembles a complete product requirements document from discovery, design, and engineering inputs.

## Framework: MoSCoW + RICE Prioritisation

### Approach

This skill uses the **MoSCoW + RICE Prioritisation** framework to structure the output.

### Sections

1. **Problem Statement**: Define the content, evidence requirements, and format for this section.
2. **Success Criteria**: Define the content, evidence requirements, and format for this section.
3. **User Stories (MoSCoW/RICE)**: Define the content, evidence requirements, and format for this section.
4. **Design Requirements**: Define the content, evidence requirements, and format for this section.
5. **Technical Constraints**: Define the content, evidence requirements, and format for this section.
6. **Scope Boundaries**: Define the content, evidence requirements, and format for this section.
7. **Traceability Matrix**: Define the content, evidence requirements, and format for this section.

### Quality Checklist

- [ ] Problem Statement completed with specific, evidence-based content
- [ ] Success Criteria completed with specific, evidence-based content
- [ ] User Stories (MoSCoW/RICE) completed with specific, evidence-based content
- [ ] Design Requirements completed with specific, evidence-based content
- [ ] Technical Constraints completed with specific, evidence-based content
- [ ] Scope Boundaries completed with specific, evidence-based content
- [ ] Traceability Matrix completed with specific, evidence-based content
