# Scoring Rubric: moat-analyzer

Analyses competitive moat and defensibility of a product, mapping moat types and surfacing vulnerabilities.

## Criteria

| # | Criterion | Weight | Description |
|---|-----------|--------|-------------|
| 1 | Moat Category Coverage | 20% | Completeness of evaluation across network effects, switching costs, data advantages, scale, brand, regulatory, and IP |
| 2 | Strength Assessment | 25% | Quality of evidence-based strength ratings per moat type |
| 3 | Vulnerability Identification | 30% | Depth of vulnerability analysis including open-source, commoditisation, and multi-homing risks |
| 4 | Reinforcement Recommendations | 25% | Actionability of moat-strengthening roadmap items prioritised by effort-to-defensibility ratio |
| **Total** | | **100%** | |

## Scale

Each criterion is scored **0-10**:
- **0**: No evidence / completely absent
- **5**: Partially present with significant gaps
- **10**: Fully present, comprehensive, no gaps

**Composite Score** = Σ (criterion score × weight)

## Grade Bands

| Grade | Composite Score | Label | Description | Recommended Action |
|-------|----------------|-------|-------------|-------------------|
| A+ | 9.0 – 10.0 | Exceptional | Exceptional performance across all criteria | Integrate reinforcement recommendations into roadmap |
| A | 8.0 – 8.9 | Strong | Strong performance across all criteria | Prioritise top 2 reinforcement items for next quarter |
| B | 7.0 – 7.9 | Good | Good performance across all criteria | Validate vulnerability assumptions before roadmap integration |
| C | 5.0 – 6.9 | Adequate | Adequate performance across all criteria | Gather competitive data to fill assessment gaps |
| D | 3.0 – 4.9 | Weak | Weak performance across all criteria | Restart with structured moat framework |
| F | 0.0 – 2.9 | Failing | Failing performance across all criteria | No defensibility analysis performed |
