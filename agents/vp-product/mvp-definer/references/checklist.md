# Checklist: mvp-definer

Defines minimum viable product scope that tests the core value hypothesis.

## Pre-Flight

- [ ] All required inputs collected and accessible
- [ ] Stakeholders identified and availability confirmed
- [ ] Timeline and deadlines established

## Execution Steps

- [ ] Step 1: Extract core value hypothesis as single testable statement
- [ ] Step 2: Map minimum user journey (story map with walking skeleton)
- [ ] Step 3: Define scope boundary: build vs fake vs omit for each step
- [ ] Step 4: Stress-test with Wizard of Oz check
- [ ] Step 5: Set success experiment with metric, threshold, and sample size
- [ ] Step 6: Negotiate scope with engineering (technical review)
- [ ] Step 7: Document MVP contract with sign-off

## Post-Completion

- [ ] All deliverables produced and stored
- [ ] Stakeholders notified of completion
- [ ] Lessons learned captured for process improvement
