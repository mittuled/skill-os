# Framework: PMM Pre-Briefing

## Core Model

A structured handoff from product to product marketing that provides the facts, framing, and context PMM needs to prepare customer-facing messaging. Delivered at least two weeks before release.

## Brief Structure

1. **Feature Facts**: Name, target release date, target audience, key capabilities, known limitations
2. **Value Narrative**: User problem solved (one sentence) + primary benefit (one sentence) + what is new vs. improved
3. **Competitive Context**: How this positions against alternatives, any competitive timing considerations
4. **Sensitivities**: Breaking changes, pricing implications, migration requirements, known gaps

## Timing Protocol

| Milestone | Action | Owner |
|-----------|--------|-------|
| T-14 days | Brief drafted and sent to PMM lead | Product Manager |
| T-10 days | PMM acknowledges receipt and flags questions | PMM Lead |
| T-7 days | PM resolves open questions, brief finalised | Product Manager |
| T-3 days | PMM confirms messaging and assets are ready | PMM Lead |

## Quality Checklist

- [ ] Brief uses customer-facing language, not engineering jargon
- [ ] Known limitations are documented alongside capabilities
- [ ] Target audience is specific (persona, segment, plan tier), not "all users"
- [ ] Value narrative focuses on the user outcome, not the feature mechanism
- [ ] Competitive context included even if the answer is "no direct competitor feature"
