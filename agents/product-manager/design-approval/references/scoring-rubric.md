# Scoring Rubric: design-approval

Reviews and approves design deliverables before engineering implementation.

## Criteria

| # | Criterion | Weight | Description |
|---|-----------|--------|-------------|
| 1 | Requirements Alignment | 35% | Coverage of acceptance criteria by design deliverable |
| 2 | Usability Assessment | 30% | Evaluation of flow consistency, pattern adherence, and user complexity |
| 3 | Decision Clarity | 20% | Clear approve/revise/reject decision with specific rationale |
| 4 | Feedback Timeliness | 15% | Decision communicated within SLA to unblock sprint |
| **Total** | | **100%** | |

## Scale

Each criterion is scored **0-10**:
- **0**: No evidence / completely absent
- **5**: Partially present with significant gaps
- **10**: Fully present, comprehensive, no gaps

**Composite Score** = Σ (criterion score × weight)

## Grade Bands

| Grade | Composite Score | Label | Description | Recommended Action |
|-------|----------------|-------|-------------|-------------------|
| A+ | 9.0 – 10.0 | Exceptional | Exceptional performance across all criteria | Design approved — engineering can proceed immediately |
| A | 8.0 – 8.9 | Strong | Strong performance across all criteria | Approved with minor notes that don't block engineering |
| B | 7.0 – 7.9 | Good | Good performance across all criteria | Revisions requested on specific criteria |
| C | 5.0 – 6.9 | Adequate | Adequate performance across all criteria | Multiple revisions needed — schedule design review |
| D | 3.0 – 4.9 | Weak | Weak performance across all criteria | Significant misalignment — rework required |
| F | 0.0 – 2.9 | Failing | Failing performance across all criteria | No design review performed |
