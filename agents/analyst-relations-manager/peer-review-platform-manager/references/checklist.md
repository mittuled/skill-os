# Checklist: Peer Review Platform Manager

Operational checklists for profile audits, review generation campaigns, and platform health monitoring.

## Checklist 1: Platform Profile Audit

Use quarterly and after any major product launch or rebrand.

### Profile Completeness (per platform)
- [ ] Product description is current and reflects the current value proposition (not an old tagline)
- [ ] All current product features are listed; deprecated features removed
- [ ] Screenshots show the current UI — no outdated interfaces
- [ ] Demo video is current (if hosted on platform)
- [ ] Integrations list reflects all active integrations
- [ ] Pricing information is accurate (or "contact for pricing" if custom)
- [ ] Categories are correct — product is listed under all relevant categories and no irrelevant ones
- [ ] Company logo and branding is current (post any rebrands)
- [ ] Profile contact / CTA link directs to a live, relevant landing page

### Profile Ranking Health
- [ ] Total review count meets or exceeds badge threshold for target badges (see framework.md)
- [ ] Review recency: >50% of reviews published in the last 6 months
- [ ] Average rating is 4.0+ on all major platforms (G2, Capterra, TrustRadius)
- [ ] No 1–2 star review has been left unresponded for >72 hours
- [ ] Competitive review volume checked: are top 3 competitors outpacing company in review velocity?
- [ ] Category placement confirmed: product is discoverable in the right category searches

## Checklist 2: Review Generation Campaign

Use before launching any review solicitation campaign.

### Audience Targeting
- [ ] Target reviewer list sourced: NPS Promoters (9–10), recently renewed, post-QBR, successful onboarding completions
- [ ] CS health scores checked for all targets — no detractors or at-risk customers included
- [ ] Customers who reviewed in the last 12 months excluded (avoid over-solicitation)
- [ ] Platform's verified purchase requirements checked: LinkedIn verification available for G2, email domain verification for Capterra
- [ ] Platform-specific ask confirmed: only asking for one platform per outreach

### Campaign Mechanics
- [ ] Personalised outreach templates prepared (not mass email blasts)
- [ ] CS team briefed on campaign: they are the primary outreach channel for highest-conversion requests
- [ ] In-app prompts configured if supported (confirm platform-specific SDK requirements)
- [ ] Any incentive programme reviewed against platform terms (no quid pro quo; post-review only)
- [ ] Direct review link tested and functional for each target platform
- [ ] Campaign timing confirmed: not launching during holiday periods or company-wide events

### Compliance Check
- [ ] No gift cards, discounts, or credits offered before review is published
- [ ] No instructions given to reviewers on what rating to leave or how to phrase feedback
- [ ] No solicitation from employees or family members of employees
- [ ] No bulk review submissions from the same IP address or device
- [ ] All outreach language is transparent about what is being requested

### Tracking Setup
- [ ] Outreach list in a tracking sheet with columns: customer name, CS owner, outreach date, platform targeted, review received (Y/N), review date
- [ ] Success metric defined: conversion rate target set per campaign
- [ ] Follow-up plan: 1 follow-up maximum at 7 days post-initial outreach

## Checklist 3: Monthly Platform Health Review

Use at the start of each month to assess platform health and plan next actions.

### Rating and Volume
- [ ] Total review count logged for each platform (G2, Capterra, TrustRadius)
- [ ] Average rating logged for each platform
- [ ] New reviews received in the last 30 days counted and logged
- [ ] All new reviews (positive and negative) identified for response
- [ ] Competitor review volumes checked (G2 Grid data available publicly)

### Sentiment Analysis
- [ ] All 1–3 star reviews from the last 30 days read and categorised by theme
- [ ] Dominant negative theme identified (if >3 reviews on same topic)
- [ ] Dominant negative theme routed to the relevant internal team (see framework.md theme routing table)
- [ ] Alert threshold checked: any single theme representing >30% of negative reviews escalated

### Response Health
- [ ] All reviews >72 hours old have received a platform response
- [ ] No copy/paste responses used across multiple reviews
- [ ] 1–2 star reviews have been responded to within 24 hours (check)
- [ ] Response tone reviewed: empathetic and solution-oriented, not defensive

### Badge and Rankings
- [ ] Current badge status logged for each platform
- [ ] Next badge milestone identified (e.g., "need 8 more reviews to reach G2 Category Leader in [segment]")
- [ ] Seasonal badge timeline noted (G2 reports publish quarterly; plan review campaigns to align)

### Campaign Planning
- [ ] Monthly review velocity vs. target assessed (see framework.md velocity targets)
- [ ] If below target: identify trigger events in the next 30 days (QBRs, renewals, onboardings) for review outreach
- [ ] Next review campaign planned if velocity is falling behind (minimum 2 reviews/month per platform target)
