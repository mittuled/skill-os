# Framework: developer-community-grower

Defines the developer community growth model, channel opportunity matrix, contributor ladder, and programme cadence for scaling developer ecosystems.

## Developer Community Growth Model

Developer communities grow through a distinct funnel. Conversion rates between stages depend on product quality and community investment.

| Stage | Definition | Typical % | Conversion Target | Key Lever |
|-------|-----------|-----------|------------------|-----------|
| Aware | Knows the product exists | 100% baseline | → 30% evaluate | Content marketing, conference presence |
| Evaluating | Reading docs, watching demos | 30% | → 40% try | Quickstart quality, sandbox access |
| User | Made first successful API call | 12% | → 50% regular | Onboarding emails, community welcome |
| Regular User | Integrates into production or project | 6% | → 20% contributor | Feature richness, community belonging |
| Contributor | Contributes code, docs, or answers | 1–2% | → 20% advocate | Recognition, contribution ladder |
| Advocate | Speaks at events, writes posts, recruits | 0.2–0.5% | Sustain | Ambassador programme, co-marketing |

## Channel Opportunity Matrix

| Channel | Developer Segment | Effort | Scalability | Best For |
|---------|------------------|--------|------------|---------|
| Blog / tutorials | All | Medium | High | SEO-driven awareness, long-term retention |
| Conference talks | Senior devs, decision-makers | High | Low | Credibility, enterprise pipeline |
| Hackathons | Hobbyists, students, early adopters | High | Medium | Rapid experimentation, community events |
| Open-source contributions | Active contributors | Medium | High | Ecosystem integration, trust signals |
| Ambassador network | Advocates in key ecosystems | Medium | Medium | Local community building, non-English markets |
| Discord / Slack community | Active users | Low–Medium | High | Real-time support, signal extraction |
| Developer newsletter | Subscribed users | Low | High | Retention, lifecycle engagement |
| Stack Overflow presence | All (via search) | Low | Very high | SEO, reputation signals |
| YouTube / video content | Visual learners, beginners | High | High | Tutorial depth, international reach |

## Contributor Ladder

Design a clear progression from consumer to contributor with specific milestones and rewards at each level.

| Level | Title | Criteria | Recognition | Benefits |
|-------|-------|---------|-------------|---------|
| 0 | User | Account created, first API call | — | Standard access |
| 1 | Active User | 3+ integrations or 30+ API calls/month | Community welcome | Priority support channel |
| 2 | Community Member | 10+ forum posts or answers | Community badge | Early feature access |
| 3 | Contributor | 1+ merged PR, doc contribution, or event presentation | Contributor badge, blog feature | Co-marketing opportunities |
| 4 | Champion | 5+ contributions across 3+ months, referrals, event speaking | Champion badge, swag, advisory seat | Advisory board, keynote speaking |
| 5 | Maintainer | Core project maintainer, sustained contributions > 12 months | Maintainer title, public recognition | Revenue share (if applicable), hiring pipeline |

## Programme Design Templates

### Hackathon ROI Formula

```
Hackathon ROI = (New signups × activation rate × LTV) − Total cost
```

Where:
- **New signups** = registrations attributable to hackathon
- **Activation rate** = % who make first API call within 7 days
- **LTV** = average revenue per converted developer over 12 months
- **Total cost** = prizes + venue/platform + team time + marketing

**Minimum viable hackathon budget benchmark:** $5–15K for virtual; $20–50K for in-person

### Ambassador Programme Structure for Developer Communities

| Component | Specification |
|-----------|-------------|
| Eligibility | Level 3+ Contributor, active in last 60 days |
| Cohort size | 10–30 per region/ecosystem |
| Term length | 6 months, renewable |
| Content commitment | 2 blog posts or talks per quarter |
| Response SLA | Acknowledge community questions within 24h |
| Recognition | Profile on website, dedicated Discord role, conference ticket budget |
| Review cadence | Monthly 1:1, quarterly cohort review |

## Success Metrics

| Metric | Benchmark | Measurement Frequency |
|--------|-----------|----------------------|
| Monthly Active Contributors (MAC) | > 1% of MAM | Monthly |
| Contributor-to-user conversion rate | > 5% | Quarterly |
| New contributor first contribution time | < 30 days from join | Monthly |
| Event attendance (% of community) | > 10% per event | Per event |
| Ambassador content output | ≥ 2 pieces/ambassador/quarter | Quarterly |
| Community-sourced signups (attribution) | > 15% of new signups | Monthly |
