# Checklist: Sales Signal Collection

## Purpose

Ensures structured, timely, and complete signal capture from every substantive prospect interaction. Use after every discovery call, demo, negotiation, or deal milestone.

## Post-Interaction Signal Capture (Within 24 Hours)

### Required Signal Fields

For every prospect interaction, capture and tag the following:

- [ ] **Interaction date and type** (discovery, demo, negotiation, reference call, etc.)
- [ ] **Attendees** (names, titles, roles in buying committee)
- [ ] **Deal stage** at time of interaction

### Signal Categories (Tag All That Apply)

#### Objections Raised
- [ ] Objection recorded verbatim (buyer's words, not paraphrase)
- [ ] Objection categorized: Pricing / Competition / Product Capability / Timing / Trust
- [ ] Objection severity tagged: Blocker / Concern / Question
- [ ] Buyer's underlying concern identified (not just surface objection)

#### Competitors Mentioned
- [ ] Competitor name(s) recorded
- [ ] Context captured: actively evaluating / previously used / mentioned by third party
- [ ] Specific competitor strengths cited by buyer
- [ ] Buyer's comparison criteria noted

#### Buying Triggers
- [ ] Trigger event identified (budget cycle, leadership change, pain event, competitor failure)
- [ ] Urgency level tagged: Immediate / This Quarter / This Year / Exploring
- [ ] Consequence of inaction stated by buyer (if mentioned)

#### Decision Process Details
- [ ] Decision-makers identified (names and titles)
- [ ] Approval steps described by buyer
- [ ] Timeline committed by buyer (specific dates, not "soon")
- [ ] Budget status noted: allocated / pending / unbudgeted

#### Deal Blockers
- [ ] Blocker described specifically
- [ ] Blocker owner identified (who can resolve it)
- [ ] Workaround discussed (if any)
- [ ] Timeline for blocker resolution

## MEDDIC Mapping

For each signal captured, map to the relevant MEDDIC element:

- [ ] **M**etrics -- Quantified business impact signals
- [ ] **E**conomic Buyer -- Budget authority and sign-off signals
- [ ] **D**ecision Criteria -- Evaluation factors and must-haves
- [ ] **D**ecision Process -- Steps, approvers, and timeline
- [ ] **I**dentify Pain -- Problem statements and impact
- [ ] **C**hampion -- Internal advocate identification and engagement level

## Context Annotation Standards

Each signal entry must include:

- [ ] What prompted the signal (question asked, topic that triggered it)
- [ ] How the prospect framed it (their words and tone)
- [ ] What it implies for deal progression (advances, stalls, or risks the deal)
- [ ] Recommended next action based on the signal

## Pattern Flagging Criteria

Flag a signal as a pattern when:

- [ ] Same signal type appears in 3+ deals within the current quarter
- [ ] Signal correlates with deal outcome (appears in closed-lost but not closed-won, or vice versa)
- [ ] Signal represents a new theme not covered in current objection handler or battle cards

### Pattern Escalation Requirements

- [ ] Deal references listed (minimum 3 deals with links to CRM records)
- [ ] Frequency count documented
- [ ] Pipeline value impacted calculated
- [ ] Recommended action proposed (update objection handler, brief Product, revise positioning)
- [ ] Escalated to Sales Manager via designated channel

## Quality Self-Check

Before submitting signal entries:

- [ ] All fields use specific language, not vague terms ("budget is $50K" not "budget exists")
- [ ] Buyer's actual words used where possible (direct quotes)
- [ ] No signals left untagged or uncategorized
- [ ] Entry completed within 24 hours of interaction
- [ ] CRM record updated (not just in personal notes)
