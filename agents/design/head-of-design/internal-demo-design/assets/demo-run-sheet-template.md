## Feedback Prompts

[Use these to structure the Q&A and prevent free-form critique derailing the session.]

1. "Does this flow solve the user problem as you understand it? What feels off?"
2. "Are there edge cases or user types this flow doesn't account for?"
3. "Does anything here create an engineering constraint we haven't discussed?"
4. "On a scale of 1-5: how confident are you we should proceed with this direction?"
