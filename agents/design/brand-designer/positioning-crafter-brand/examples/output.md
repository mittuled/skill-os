# Brand Visual Direction: Flowline

| Field | Value |
|-------|-------|
| Date | 2026-03-31 |
| Brand Designer | Brand Designer |
| Brand | Flowline |
| Positioning | Fastest CI/CD platform for engineering teams who care about quality as much as speed |
| Key Attributes | Precision, Speed, Trust |
| Reference Brands | Linear, Vercel, Stripe Docs |
| Skill | positioning-crafter-brand |

## Visual Direction Summary

The Flowline visual language bridges precision and speed: geometric structure with dynamic energy. The design should feel like a tool built by engineers, for engineers — not a marketing brochure.

## Primary Visual Cues

From the positioning attributes (precision + speed + trust):
1. **Geometric shapes** (precision)
2. **Monospace type** for code/technical contexts (precision)
3. **Grid-based layouts** (precision)
4. **Horizontal motion** — diagonal elements, directional flow (speed)
5. **Clean whitespace** — conveys quality and control (trust)

## Colour Families

| Attribute | Colour Direction |
|-----------|-----------------|
| Precision | Blues, greys, blacks |
| Speed | Electric blues, vibrant accent |
| Trust | Navy, white, muted tones |

**Combined direction**: Deep navy/black foundation, electric blue as the primary action colour, with generous white space. Accent is used sparingly for speed/energy.

## Photography and Illustration

| Asset Type | Direction |
|-----------|-----------|
| Photography | Product screenshots as primary visuals — real product builds trust with engineers |
| Illustration | Minimal line illustration only if product screenshots are unavailable |
| Iconography | Filled icons, 2px corner radius, 24px base size |
| Stock photography | Banned — no stock photos of people typing |

## Spacing Philosophy

Generous whitespace between elements to convey quality and reduce cognitive load. Engineers expect information-dense UIs inside the product; marketing surfaces should breathe.

## Design Do/Don't

| Do | Don't |
|----|-------|
| Use product screenshots as the primary visual | Use abstract metaphors engineers won't relate to |
| Use dense information layouts where appropriate | Use rounded, playful design language |
| Bold sans-serif with clear hierarchy | Use stock photography of people typing on laptops |
