# Framework: Launch Narrative Brand

Defines the structural framework for building visual launch narratives that carry a unified brand story across every launch touchpoint.

## Narrative Architecture

### Story Arc Model

| Phase | Narrative Beat | Visual Role | Asset Types |
|-------|---------------|-------------|-------------|
| Problem | Audience tension or status quo | Contrast; dark/heavy to signal friction | Hero problem visual, social teaser |
| Transformation | The shift the product enables | Motion or transition metaphor | Hero key visual, animated loop |
| Outcome | The world after the product | Aspirational; light/open | Product screenshots, testimonial visuals |

### Visual Consistency Rules

1. **Hero concept lock**: All channel assets derive from a single hero visual concept. No channel gets a unique concept.
2. **Colour invariance**: Launch palette must be a subset of the brand identity palette. No launch-exclusive colours.
3. **Typography fidelity**: Only approved typefaces from the brand identity system. Weight and scale variations are permitted.
4. **Motion character**: Motion direction, easing, and speed must match the brand personality (e.g., a precise brand uses ease-in-out, not spring physics).

### Channel Adaptation Table

| Channel | Format | Key Constraint | Narrative Priority |
|---------|--------|---------------|-------------------|
| Social feed (1:1) | 1080×1080px | 3-second attention capture | Transformation visual |
| Social stories (9:16) | 1080×1920px | Text-safe zone top/bottom 15% | Outcome with CTA |
| Email header | 600px wide | Static only (email client limits) | Problem or outcome |
| OG image | 1200×630px | Text legible at 300px width | Brand + launch name |
| Landing page hero | Full-bleed | Responsive: 1440/768/375px | Full arc condensed |
| Presentation title | 16:9 | Projector contrast (min 4.5:1) | Transformation |
| Paid display | IAB standard sizes | File size < 150KB | Outcome + CTA |

## Visual Narrative Principles

1. **Earned complexity**: Reserve the most visually complex asset for the highest-attention moment (launch day hero). Simpler assets build up to it.
2. **Progressive disclosure across channels**: Users who see the teaser, then the hero, then the detail assets should experience the story as a coherent sequence.
3. **Identity amplification**: A launch should feel like the most expressive version of the brand identity, not a departure from it.
4. **Emotion before information**: The first visual impression communicates the emotional promise; copy carries the functional detail.
