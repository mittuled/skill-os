# Checklist: cap-table-manager

Covers the recurring tasks required to keep the cap table accurate, complete, and audit-ready across equity events, fundraising, and reporting cycles.

## Equity Event Recording

### For Every Equity Grant (Options, RSUs, Restricted Stock)

- [ ] Board resolution authorizing the grant is signed and on file
- [ ] Grant agreement is signed by the employee and the company
- [ ] Grant date matches the board authorization date (not the employee start date)
- [ ] Exercise price matches the most recent 409A FMV at grant date
- [ ] Vesting schedule entered: start date, cliff date, monthly vesting schedule, and any performance conditions
- [ ] Acceleration terms documented (single-trigger, double-trigger, or none)
- [ ] Grant recorded in the cap table platform (Carta, Pulley, or equivalent)
- [ ] Confirmation email or equity statement sent to the grantee

### For Every Option Exercise

- [ ] Exercise request received in writing from the option holder
- [ ] Exercise price calculated: number of shares × exercise price per share
- [ ] Payment method confirmed (cash, cashless, same-day sale)
- [ ] Payment received and confirmed (bank transfer or brokerage confirmation)
- [ ] Shares transferred from option plan to issued common shares in cap table
- [ ] Post-exercise 409A implications reviewed (does the company need an updated valuation?)
- [ ] Tax withholding obligations assessed for NSOs (FICA, federal, state)
- [ ] Option holder notified with confirmation of share issuance

### For Terminations With Equity Implications

- [ ] Last day of employment confirmed with HR
- [ ] Post-termination exercise window calculated per grant agreement (typically 90 days for standard ISOs)
- [ ] Unvested shares cancelled and returned to option pool
- [ ] Option holder notified of exercise deadline in writing
- [ ] If no exercise by deadline: options cancelled, pool replenished, cap table updated
- [ ] ROFR rights on any already-exercised shares reviewed per governing documents

## SAFE and Convertible Note Management

- [ ] New SAFE or note registered in the convertible instrument register immediately upon closing
- [ ] Key terms documented: investor, amount, date, valuation cap, discount rate, MFN status, pro-rata rights
- [ ] MFN provisions checked: does this SAFE trigger MFN obligations on earlier SAFEs?
- [ ] SAFE register reconciled with bank records (each SAFE should match a wire receipt)
- [ ] Conversion scenarios modelled at 1x, 1.5x, and 2x the valuation cap
- [ ] Total outstanding SAFE/note exposure calculated as % of fully-diluted shares

## Quarterly Reconciliation

- [ ] Cap table system reconciled against the stock ledger
- [ ] All equity events since last reconciliation verified against board resolutions
- [ ] Total issued shares verified against the certificate of incorporation (authorized share limit)
- [ ] Total authorized shares minus issued shares = available headroom (flag if headroom <20%)
- [ ] Option pool utilization calculated: (granted + exercised) / total authorized pool
- [ ] Convertible instruments included in fully-diluted share count
- [ ] Any discrepancies documented and routed to legal for resolution
- [ ] Reconciliation sign-off obtained from CFO

## 409A Coordination

- [ ] 409A valuation current — check if more than 12 months old or if a material event occurred since last valuation
- [ ] Material events that trigger a new 409A: fundraising close, 30%+ revenue change, strategic shift, M&A inquiry
- [ ] All option grants since last 409A use the correct FMV exercise price
- [ ] 409A firm engaged if refresh is needed; deliverable timeline confirmed
- [ ] New 409A distributed to the board for review before next option grant cycle

## Dilution Modelling (Pre-Transaction)

- [ ] Model created for the specific transaction (priced round, SAFE issuance, option pool expansion)
- [ ] All existing convertible instruments included in conversion calculations
- [ ] Option pool shuffle modelled (pool expansion counted in pre-money for investors)
- [ ] Pre-money and post-money ownership tables complete for all share classes
- [ ] Waterfall analysis run at 1x, 2x, 3x, 5x exit valuations
- [ ] Liquidation preference stack documented (participating vs. non-participating, multiples)
- [ ] Output reviewed by CFO before presenting to board or investors

## Reporting and Access

- [ ] Quarterly ownership summary distributed to all shareholders with information rights
- [ ] Individual equity statements available to all option holders on request (not proactively — reduces noise)
- [ ] Access controls reviewed: founders have full access; investors have read-only (their position only unless info rights cover full cap table); employees see only their own grants
- [ ] Any cap table exported outside the platform logged with recipient and date
