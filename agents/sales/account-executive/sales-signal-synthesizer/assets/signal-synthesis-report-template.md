## Executive Summary

[2-3 sentences summarizing the top 3 themes, total pipeline value affected, and the highest-priority recommendation.
GUIDANCE: Lead with the most impactful theme and its revenue consequence. Do not describe the synthesis process.]

## Signal Dataset Overview

[Summary statistics for the analysis period.

GUIDANCE:
- Good: Table with analysis period, total signals analyzed, signals by type (objections, competitive mentions, feature requests, buying triggers, blockers), number of unique deals represented, total pipeline value represented
- Bad: "We looked at a lot of signals"
- Format: Summary statistics table with counts and percentages]

## Theme Analysis

[Top 5 themes by frequency and deal-value impact.

GUIDANCE:
- Good: One subsection per theme with: theme name, frequency (N deals, N mentions), underlying buyer concern, 2-3 representative quotes (anonymized), affected deal stages, and connection to win/loss outcome
- Bad: "Pricing is a common theme"
- Format: Structured subsection per theme with consistent fields. Order by revenue impact (highest first).]

## Impact Scorecards

[Revenue impact quantification per theme.

GUIDANCE:
- Good: Table with theme name, total pipeline value affected, number of deals, average deal size, win rate for deals with this theme vs. without, average cycle time impact, and revenue-weighted priority score
- Bad: "These themes are important"
- Format: Table sorted by revenue-weighted priority score. Include formula for priority score calculation.]

## Actionable Recommendations

[Specific actions per theme directed to the responsible team.

GUIDANCE:
- Good: Table with theme, recommended action, responsible team (Product/Marketing/Sales/SE), urgency (P1/P2/P3), expected pipeline impact if addressed, and deadline
- Bad: "Product should fix this"
- Format: One row per recommendation. Each action must be specific enough to be assigned as a task.]

## Recommendations

[Cross-cutting recommendations for improving signal quality and synthesis cadence.

GUIDANCE: Each recommendation should be:
- Specific (not "collect more signals" but "add required CRM field for competitor mention at qualification stage")
- Actionable (assignable to a person/team)
- Prioritized (P1/P2/P3)]
