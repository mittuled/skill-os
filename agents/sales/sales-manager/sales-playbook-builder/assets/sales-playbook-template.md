## ICP and Buyer Personas

[Target customer profile and buying committee.

GUIDANCE:
- Good: ICP table (company size, industry, tech stack, budget range, buying trigger) plus persona table (role, title, priority, pain points, messaging hook, preferred channel) with minimum 4 personas
- Bad: "Our target customer is mid-market companies"
- Format: Two tables -- one for ICP firmographics, one for buyer personas]

## Qualification Framework

[MEDDIC or BANT criteria with scoring.

GUIDANCE:
- Good: Table with each element, definition, discovery questions (2-3 per element), "what good looks like" examples, red flags, and scoring (0-3). Include minimum advancement thresholds per stage.
- Bad: "Use MEDDIC to qualify deals"
- Format: One subsection per MEDDIC element with structured content. Reference: `references/framework.md`]

## Discovery Question Bank

[Structured questions organized by persona and stage.

GUIDANCE:
- Good: Questions grouped by buyer persona, then by discovery phase (pain, impact, decision process, timeline). Each question tagged with the MEDDIC element it uncovers. Include "listen for" guidance after each question.
- Bad: "Ask good discovery questions"
- Format: Table per persona with columns: Phase, Question, MEDDIC Element, Listen For]

## Competitive Battle Cards

[Positioning against top 3 alternatives.

GUIDANCE:
- Good: One card per competitor with: positioning summary, strengths (be honest), weaknesses (be specific), landmines to set in discovery, trap questions, "why we win" (with deal evidence), "why we lose" (with deal evidence), and counter-positioning talk track
- Bad: "We are better than competitors"
- Format: One subsection per competitor with consistent structure]

## Objection Handler

[Top 15 objections with response frameworks.

GUIDANCE:
- Good: Table with objection, category (pricing/competition/product/timing/trust), frequency (high/medium), response using acknowledge-reframe-evidence pattern, and proof point
- Bad: "Handle objections effectively"
- Format: Table sorted by frequency, with response column using the A-R-E pattern]

## Closing Playbook

[Techniques matched to deal types.

GUIDANCE:
- Good: Trial close questions per stage, proposal framework (what to include, what to exclude), negotiation guardrails (discount limits, approval paths), and mutual action plan template with milestone types
- Bad: "Close deals efficiently"
- Format: Subsections for trial closes, proposal framework, negotiation rules, and mutual action plan template]

## Recommendations

[Playbook rollout and maintenance plan.

GUIDANCE: Each recommendation should be:
- Specific (not "train the team" but "run 90-minute playbook walkthrough with all AEs by [date]")
- Actionable (assignable to a person/team)
- Prioritized (P1/P2/P3)
- Include update cadence (quarterly refresh, trigger-based updates for competitive changes)]
