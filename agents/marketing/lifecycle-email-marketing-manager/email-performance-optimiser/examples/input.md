# Scenario: Q3 Email Optimisation Programme Review — B2B SaaS Onboarding

Evaluate the email optimisation programme for a B2B SaaS company's onboarding sequence. The team has been running A/B tests for the past quarter across a 7-email onboarding flow with ~2,000 new signups per month.

## Programme Details

- **Sequence**: 7-email onboarding flow triggered by free trial signup
- **Monthly test volume**: 2-3 A/B tests per month
- **Platform**: Customer.io with built-in A/B test support
- **Primary metric**: Trial-to-paid conversion rate (baseline: 8.2%)
- **Secondary metrics**: Open rate, click-through rate, activation milestone completion

## Tests Run This Quarter

1. **Welcome email subject line**: Tested "Welcome to Acme" vs "Your first project is waiting" — the second won with 34% higher open rate (p < 0.01, n=1,800)
2. **Day-3 email CTA colour**: Tested blue vs green button — no significant difference after 2,200 sends (p = 0.43)
3. **Day-5 email send time**: Tested 9am vs 2pm local time — 2pm showed 12% higher click rate (p = 0.03, n=1,600)
4. **Day-7 email content**: Tested feature list vs customer story format — customer story had 22% higher activation click-through but test was called at n=400
5. **Subject line emoji**: Tested with/without emoji on day-2 reminder — emoji version showed 8% lift in opens (p = 0.08, n=1,200)

## Current State

- Test hypotheses are documented in a Notion page before launch
- Results are shared in the weekly marketing standup but not archived systematically
- No downstream revenue tracking per test — only email-level metrics
- The team does not reference prior test results when designing new experiments
