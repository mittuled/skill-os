# Scoring Rubric: backlog-groomer

Refines, estimates, and re-prioritises backlog items so upcoming sprints contain only sprint-ready stories.

## Criteria

| # | Criterion | Weight | Description |
|---|-----------|--------|-------------|
| 1 | Acceptance Criteria Quality | 25% | Presence of Given/When/Then acceptance criteria on every story in sprint horizon |
| 2 | Estimation Accuracy | 25% | Story-point estimates agreed with delivery team using consistent methodology |
| 3 | RICE Prioritisation | 25% | Stories ranked using RICE framework with documented Reach, Impact, Confidence, and Effort |
| 4 | Backlog Hygiene | 25% | Stale stories archived, duplicates merged, dependency flags routed |
| **Total** | | **100%** | |

## Scale

Each criterion is scored **0-10**:
- **0**: No evidence / completely absent
- **5**: Partially present with significant gaps
- **10**: Fully present, comprehensive, no gaps

**Composite Score** = Σ (criterion score × weight)

## Grade Bands

| Grade | Composite Score | Label | Description | Recommended Action |
|-------|----------------|-------|-------------|-------------------|
| A+ | 9.0 – 10.0 | Exceptional | Exceptional performance across all criteria | Backlog is sprint-ready — proceed to sprint planning |
| A | 8.0 – 8.9 | Strong | Strong performance across all criteria | Minor refinements then proceed to sprint planning |
| B | 7.0 – 7.9 | Good | Good performance across all criteria | Address flagged stories before sprint planning |
| C | 5.0 – 6.9 | Adequate | Adequate performance across all criteria | Schedule additional grooming session |
| D | 3.0 – 4.9 | Weak | Weak performance across all criteria | Restart grooming with engineering present |
| F | 0.0 – 2.9 | Failing | Failing performance across all criteria | No meaningful grooming performed |
