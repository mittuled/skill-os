# Framework: Sales Playbook Messaging

Reference framework for structuring the messaging components of a sales playbook.

## Messaging Framework Architecture

```
Sales Playbook Messaging
├── Elevator Pitch (30 seconds)
├── Persona-Specific Value Props
│   ├── Economic Buyer
│   ├── Technical Evaluator
│   └── End User Champion
├── Discovery Questions (per persona)
├── Objection Handlers
├── Competitive Differentiators (per competitor)
└── Closing Reinforcements
```

## Persona Messaging Matrix

| Element | Economic Buyer | Technical Evaluator | End User Champion |
|---------|---------------|-------------------|------------------|
| Primary concern | ROI, risk, strategic alignment | Integration, performance, security | Daily workflow, ease of use, time savings |
| Value prop angle | Business outcome, competitive advantage | Technical capability, architecture fit | Productivity gain, friction removal |
| Language register | Strategic, financial | Technical, specific | Practical, relatable |
| Proof point type | Revenue/cost metrics, logo references | Architecture diagrams, performance benchmarks | User testimonials, workflow comparisons |
| Discovery question focus | "What does success look like?" | "How does this fit your stack?" | "Walk me through your current workflow" |

## Objection Handler Template

For each common objection:

| Component | Description | Example |
|-----------|-------------|---------|
| Objection | Exact words the buyer uses | "Your competitor is 30% cheaper" |
| Acknowledge | Show you heard and understand | "Price is an important consideration" |
| Reframe | Shift the conversation to value | "Let's look at total cost of ownership including implementation time" |
| Proof | Evidence that supports the reframe | "Customer X calculated a 6-month payback period because..." |
| Bridge | Return to your value proposition | "So the question is really about time-to-value, and that's where we..." |

### Objection Categories
| Category | Trigger | PMM Response Strategy |
|----------|---------|---------------------|
| Price | "Too expensive", "competitor is cheaper" | Reframe on value per outcome, not cost per seat |
| Timing | "Not now", "maybe next quarter" | Create urgency with cost-of-delay or competitive risk |
| Competition | "We're evaluating [competitor]" | Lean into differentiators; never disparage |
| Risk | "This seems risky", "you're too new" | Social proof (logos, case studies, security certifications) |
| Status quo | "What we have works fine" | Quantify the hidden cost of the current approach |
| Authority | "I need to get approval" | Equip the champion with an internal business case |

## Competitive Differentiator Block Template

For each primary competitor:

```
## vs. [Competitor Name]

### Positioning
[One sentence: how we are fundamentally different from this competitor]

### What to Say
- [Specific claim with evidence]
- [Specific claim with evidence]
- [Specific claim with evidence]

### What NOT to Say
- [Statement to avoid and why]
- [Statement to avoid and why]

### When They Win
[The buying scenario where this competitor has an advantage — so reps can qualify early]

### When We Win
[The buying scenario where we have the advantage — so reps can steer the conversation]
```

## Discovery Question Design Principles

1. **Open-ended**: Start with "how", "what", or "tell me about" — not yes/no
2. **Pain-oriented**: Guide the conversation toward problems your product solves
3. **Persona-specific**: Different questions for different buying roles
4. **Progressive depth**: Start broad ("What are your top priorities?"), then narrow ("How do you currently handle X?")
5. **Diagnostic**: Help the rep assess fit, not just build rapport

## Messaging Versioning

| Element | Description |
|---------|-------------|
| Version tag | Date stamp on every messaging section (e.g., "Updated: 2026-03-15") |
| Change log | Brief note on what changed and why |
| Review cadence | Full refresh quarterly; interim update after positioning change or competitive shift |
| Validation | Every update reviewed by 2-3 experienced AEs before publication |
