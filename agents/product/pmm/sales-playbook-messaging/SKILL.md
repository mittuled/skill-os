# sales-playbook-messaging

## Agent: PMM
L2 product marketing manager responsible for competitive intelligence, positioning, GTM planning, pricing strategy, launch narrative, and sales enablement.

Department ethos: [ideal-product.md](../../../../departments/product/ideal-product.md)

## Skill Description
Contributes the messaging framework to the sales playbook, providing the structured talk tracks, persona-specific value propositions, objection handlers, and competitive differentiators that reps use in every stage of the deal cycle.

## When to Use
- Building a new sales playbook for a product launch or new market segment.
- Refreshing the messaging sections of an existing playbook after a positioning update.
- Win/loss analysis reveals that reps are not articulating differentiation effectively.
- A new persona or buying centre role needs to be added to the playbook with tailored messaging.
- Competitive landscape has shifted and the playbook's competitive sections need updating.

## Workflow
1. Start from the positioning document and messaging hierarchy. The playbook messaging must be a direct translation of positioning into sales-ready language, not an independent creation.
2. Define the messaging framework structure: value proposition by persona, elevator pitch, discovery questions, objection handlers, competitive differentiators, and closing reinforcements.
3. Write persona-specific value propositions. Each buyer persona (economic buyer, technical evaluator, end user champion) needs messaging that speaks to their specific priorities and evaluation criteria.
4. Draft discovery questions mapped to each persona. Questions should surface the pains that the product's positioning addresses, guiding the conversation toward your strengths.
5. Build objection handlers. For each common objection, provide: the objection as the buyer states it, the recommended reframe, a proof point or customer example, and a bridge back to the value proposition.
6. Write competitive differentiator blocks. For each primary competitor, provide a positioning statement (how we differ), a "what to say" section, and a "what not to say" section.
7. Add closing reinforcements: summary statements reps use in proposal or negotiation stages to reiterate why the buyer should choose now.
8. Review with two to three experienced AEs. Test whether the messaging matches how they actually sell and where it misses.
9. Incorporate feedback and publish the messaging sections into the sales playbook. Tag each section with a version date so reps know when it was last updated.

## Anti-Patterns
- **Writing messaging in marketing language instead of sales language.** *Why: Reps will not use messaging that sounds like ad copy; playbook language must feel natural in a conversation, not a campaign.*
- **Creating the messaging framework without AE input.** *Why: PMM understands positioning but AEs understand the deal cycle; messaging that ignores how deals actually unfold will be ignored.*
- **Providing a single generic value prop instead of persona-specific messaging.** *Why: An economic buyer and a technical evaluator care about different things; one-size-fits-all messaging resonates with no one.*
- **Omitting competitive differentiators.** *Why: Reps face competitors in nearly every deal; a playbook without competitive messaging forces improvisation at the moment it matters most.*

## Output

**Success:**
- Messaging framework integrated into the sales playbook with persona-specific value props, discovery questions, objection handlers, competitive differentiators, and closing reinforcements.
- Validated by experienced AEs and tagged with a version date.
- Reps reference the messaging in deal prep and call recordings.

**Failure:**
- Messaging sections written in marketing language that reps do not use in actual conversations.
- Generic value proposition without persona or deal-stage specificity.
- No version date, leaving reps unsure whether the messaging reflects current positioning.
